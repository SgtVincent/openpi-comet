# 项目规则
1. 对于 openpi-comet 的代码和脚本，请永远先运行 `source .venv/bin/activate` 激活uv 环境再执行，警惕代码运行中执行 .venv 以外的python 依赖
2. 请尽量用户友好，在执行任务的过程中多总结你的操作步骤和结果，说明你发现了什么，你想做什么，你接下来要做什么
3. 对于代码的新增保持一致的代码风格，保持代码的可读性和可维护性，遵守最小化修改原则
4. 对于复杂任务，尽可能先规划好实现步骤，再开始实现，不要直接开始实现，否则会导致代码的不可维护性
5. 单步可以完成的简单任务，直接在命令行中执行，不要写脚本，否则会导致代码的不可维护性
6. **资源利用与 NAS 优化**：当前环境为 110 CPU + 8 GPU，且数据存储在 NAS 上。为缓解 IO 瓶颈，训练时请尽可能调大 dataloader 的 `num_workers`（推荐单卡测试 16+，多卡训练每卡 10+），避免使用 `num_workers=0`。
